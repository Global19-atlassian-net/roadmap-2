Le futur radieux de io.js

Une feuille de route dirigée par la communauté

Pour la communauté, par la communauté

io.js est une version alternative de node.js et compatible avec la quasi-totalité des modules dans npm.

io.js est un projet open source dirigé par la communauté. Il a été lancé par les principaux contributeurs de node.js mais dans les quelques premiers mois a attiré plus de développeurs actifs que le projet node.js n’en a jamais eu dans toute son histoire.

Qu'attendent les gens de io.js?

  1. Stabilité
  2. Transparence (débogage et traçage)
  3. Meuilleur flux
  4. Support à long terme
  5. Adopter des normes en cours
  6. Localisation

Liste de préoccupations consolidée à partir du projet feuille de route. Les contributeurs sont les bienvenus!

Politique de stabilité

Nous avons le plus grand écosystème de paquet de n'importe quelle langage, nous ne pouvons pas le casser.

  • Nous ne supprimons pas les API JS.
  • Versionage sémantique
    • Tout ajout sur l'API signifie une incrémentation mineure de version.
    • Les changements dans l'API C ++ de V8 qui peuvent être gérés par nan entraînent une incrémentation mineure de version, et ceux qui ne le peuvent pas entraînent une incrémentation majeur.

Depuis la section de politique de stabilité de la feuille de route. Les contributeurs sont les bienvenus!

NG (Génération moderne)

  • Une nouvelle plate-forme pour les applications modernes JavaScript.
  • Restera compatible en amont.
  • Les nouvelles API peuvent être centrée sur ES6/7 sans casser les anciens modules et API.
    • li>
  • Sépare les API à venir avenir des améliorations à la plateforme actuelle.

Les discussions NG sont en cours sur le dépôt NG.

Débogage & Traçage

Le noyau devrait permettre un écosystème massif et diversifié d’outils de débogage et de traçage plutôt qu’entériner des abstractions de haut niveau comme les domaines.

  • Intégrer avec trace_event (Nouvelle API de traçage pour V8 et Chrome).
  • Documenter et évangéliser un écosystème de débogage.
  • Unifier les points d’accès au système de traçage (DTrace, LTTng, etc.)

Ce travail est piloté par le groupe de travail sur le Traçage. Les contributeurs sont les bienvenus!

Flux

  • Le groupe de travail sur les flux a repris la main pour readable-stream, qui est maintenant le io.js module de flux officielles.
  • Corriger tous les problèmes de compatibilité existants.
  • Simplifier la création de flux pour éviter les erreurs de l'utilisateur.
  • Explorer les interfaces WHATWG Streams pour identifier les problèmes de compatibilité.
  • Améliorer les performances de flux.

Ce travail est piloté par le équipe de travail des flux. Les contributeurs sont les bienvenus!

Support à long terme

  • De nouvelles lignes de release seront livrés avec dépendances stables et maintenus.
  • Releases de correctifs pour les anciennes versions seront publiées aussi longtemps que les gens contribuent activement.
  • io.js aura la responsabilité de vieilles versions de dépendances comme V8 qui apparaissent dans les versions précédentes lorsque leurs responsables cesser de les maintenir.

Résumé issu de la politique de stabilité. Les contributeurs sont les bienvenus!

Localisation

  • Réduire l’inclusion de l'icu (taille du binaire et occupation mémoire) .
  • Favoriser des communauté de localisation dynamique.
    • Les communautés de langue autonome (plus de 30 enregistrées jusqu'à présent).
    • Efforts d'évangélisation pour développer l'adhésion (plus de 160 membres à ce jour).
    • Améliorer l'outil pour traduire le site web, les documentations API, les installeurs et les présentations.

Vous pouvez vous participer en rejoignant la communauté pour votre langue :)
