The Bright Future of io.js

Une feuille de route dirigé par la communauté

Pour la communauté, par la communauté

io.js est une plateforme basée sur node.js et compatible avec npm.

io.js est un projet open source dirigé par la communauté. Il a été lancé par les principaux contributeurs de node.js mais dans les quelques premiers mois a attiré des développeurs plus actifs que le projet node.js a jamais eu dans toute son histoire.

Qu'est-ce que les gens veulent de io.js?

  1. Stabilité
  2. Transparence (débogage et traçage)
  3. Meuilleur streams
  4. Support à long term
  5. Adopter des normes en cours
  6. Localisation

Concerns list compiled from du projet roadmap. Les contributeurs sont les bienvenues!

Politique de stabilité

Nous avons le plus grand écosystème de paquet de n'importe quelle langue, nous ne pouvons pas le détruire.

  • Nous ne supprimons pas les API JS.
  • Versionage sémantique
    • Tout ajout sur l'API signifie une incrementation mineur de version.
    • Les changements dans l'API C ++ de V8 de traités par nan une incrementation mineur de version et une incrementation majeur de version si on nne peut pas.

Depuis la section de politique de stabilité de la roadmap. Les contributeurs sont les bienvenues!

Chemin

Release Toute les versions build.
Doit être stable et prêt pour la production.
Canary Nightly builds avec Canary V8 et d'autres nouvelles fonctionnalités.
NG Génération suivante ES6/7 centric API.

NG (Next Generation)

  • Une nouvelle plate-forme pour les applications modernes JavaScript.
  • Restera compatible indéfiniment.
  • Nouvelles API peuvent être centrée sur ES6/7 sans casser les anciens modules et API.
    • li>
  • Séparer les travaux sur API avenir des améliorations à la plateforme actuelle.

Les discussions NG sur le repo NG.

Débogage & Tracing

Base devrait permettre un écosystème massive et diversifiée de outils de débogage et de traçage plutôt que entériner abstractions de haut niveau comme les domaines.

  • Intégrer avec trace_event (Nouvelle API de traçage pour V8 et Chrome).
  • Documenter et évangéliser un écosystème de débogage.
  • Système de traçage Unify critère (DTrace, LTTng, etc)

Le travail étant dirigée par l' équipe de travail de traçage. Les contributeurs sont les bienvenues!

Streams

  • L'équipe de travail des streams a pris fin pour readable-stream, qui est maintenant le io.js module de stream officielles.
  • Corriger tous les problèmes de compatibilité existants.
  • Simplifier la création de stream pour éviter les erreurs de l'utilisateur.
  • Explorer les WHATWG Streams pour interfacer et d'identifier les problèmes de compatibilité.
  • Améliorer les performances de stream.

Work being lead by the Streams Working Group. Les contributeurs sont les bienvenues!

Support à long term

  • De nouvelles lignes de release seront livrés avec dépendances stables et soutenus.
  • Releases de patch d'anciennes versions seront publiées aussi longtemps que les gens contribuent activement.
  • io.js aura la responsabilité de vieilles versions de dépendances comme V8 qui apparaissent dans les versions précédentes lorsque leurs responsables cesser de les soutenir.

Sommaire de la politique de stabilité. Les contributeurs sont les bienvenues!

Localisation

  • Réduire l'impact de l'inclusion icu (empreinte et la mémoire binaire).
  • Favoriser une communauté de localisation dynamique.
    • Les communautés de langue autonome (plus de 30 enregistrés jusqu'à présent).
    • Efforts d'évangélisation pour développer l'adhésion (plus de 160 membres à ce jour).
    • Améliorer l'outil pour traduire un site web, documentation de l'API, les installateurs et les diapositives.

Vous pouvez vous impliquer en rejoindre la communauté pour votre langue :)
